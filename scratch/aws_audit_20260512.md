## 4. 네트워크 / 보안

| 포트 | 상태 | 평가 |
| :--- | :--- | :--- |
| 22 (SSH) | LISTEN 0.0.0.0 | ✅ |
| 80 (HTTP) | LISTEN 0.0.0.0 (nginx) | ✅ |
| **443 (HTTPS)** | **미가동** | ⚠️ 결제 심사 통과 불가 |
| **8000 (Backend)** | **LISTEN 0.0.0.0 (외부 노출)** | ⚠️ 보안 위험 — 127.0.0.1로 제한 필요 |

**중요한 보안 이슈**:
- 백엔드(8000)가 인터넷에 직접 노출되어 Nginx를 우회 가능. uvicorn 로그에 중국·미국 등 다양한 IP의 스캐너가 계속 GET / 시도 중 ("Invalid HTTP request received" 다수). 위험 수준은 아직 높지 않지만 지금 수정하는 게 정석.
- 조치: ① 보안그룹에서 8000 차단 ② uvicorn `--host 127.0.0.1`로 변경

**Nginx 설정** (`/etc/nginx/sites-available/washercrm`):
- `server_name _;` (모든 호스트 매칭) — 도메인 확보 후 `whiteon.kr`로 명시 권장
- `location /api/v1` → `proxy_pass http://127.0.0.1:8000` ✅
- SSL 블록 없음 — Phase A §2.3에서 추가
- HTTP → HTTPS 강제 리다이렉트 없음
