# 📌 Antigravity Coding Rules (Harness)

### 1. 코드 보존 원칙 (CRITICAL)
- **기존 완성 코드 보존**: 이미 구현되어 정상 작동하는 기능(필터, 검색, 정산 로직, 탭 등)은 UI 고도화 시에도 절대 삭제하거나 수정하지 않는다.
- **최소 침습 수정**: 새로운 요구사항이 있을 경우, 기존 로직을 건드리지 않고 새로운 코드를 덧붙이는 방식으로 작업한다.
- **기능 유효성 검증**: UI 레이아웃을 바꿀 때 반드시 이전 버전의 모든 `id`와 `onclick` 함수들이 유지되는지 확인한다.

### 2. UI/UX 디자인 원칙
- **프리미엄 디자인 유지**: WhiteOn 브랜드 아이덴티티(파랑/슬레이트 톤)와 고품질 애니메이션을 유지한다.
- **데이터 가시성**: 관제 화면에서는 데이터의 상세함이 디자인보다 우선한다.

### 3. 백엔드 호환성
- **하위 호환성 유지**: 새로운 필드를 추가할 때 기존 데이터(Null 값이 있는 구버전 데이터)와 충돌하지 않도록 방어 코드를 작성한다.
