# 🖥️ WasherCRM 백엔드 서버 관리 가이드

본 문서는 WasherCRM 백엔드 서버를 처음 운영하는 관리자를 위한 상세 매뉴얼입니다. 순서대로 따라 하시면 누구나 서버를 기동하고 테스트할 수 있습니다.

---

## 1. 서버 시작하기 (Startup)

서버를 켜기 위해 윈도우의 터미널(명령 프롬프트/CMD)을 열고 아래 순서대로 입력하세요.

### Step 1: 프로젝트 폴더로 이동
```cmd
cd C:\Users\zpwkg\Documents\WasherCRM\backend
```

### Step 2: 경로 설정 (중요!)
파이썬이 프로젝트 내부 파일을 찾을 수 있게 경로를 등록합니다.
```cmd
set PYTHONPATH=.
```

### Step 3: 서버 실행
아래 명령어를 입력하면 서버가 기동됩니다.
```powershell
$env:PYTHONPATH="."
# 내부 테스트용 (나만 보기)
py -m uvicorn app.main:app --reload --port 8888

# 외부 테스트용 (아버님/지인 공유용)
py -m uvicorn app.main:app --reload --port 8888 --host 0.0.0.0
```
*   `--reload`: 코드를 수정하면 서버가 자동으로 재시작되는 옵션입니다.
*   `--port 8888`: 윈도우 시스템 포트 충돌을 피하기 위해 8888번 포트를 사용합니다.

## 3. Swagger UI 상세 가이드 (API 마스터하기)

`http://127.0.0.1:8888/docs`는 단순히 문서를 보는 곳이 아니라, **서버와 직접 대화하는 창구**입니다.

### 3.1 API 색상의 의미
- <span style="color: #49cc90">**POST (초록)**</span>: **생성** (새로운 식당, 지사, 접수 건을 만들 때)
- <span style="color: #61affe">**GET (파랑)**</span>: **조회** (데이터 목록을 보거나 상세 내용을 확인할 때)
- <span style="color: #fca130">**PATCH (주황)**</span>: **일부 수정** (상태 변경, 정보 수정 등)
- <span style="color: #f93e3e">**DELETE (빨강)**</span>: **삭제** (데이터를 지울 때 - 주의 필요!)

### 3.2 추천 테스트 시나리오 (데이터 쌓기 순서)
데이터베이스가 비어있을 때는 아래 순서대로 테스트해야 에러가 나지 않습니다.

1.  **식당 등록** (`POST /api/v1/restaurants/`): 
    - 식당 정보를 입력하고 실행합니다. 결과에서 나온 **`id`**를 복사하세요.
    - > [!WARNING]
    - > Swagger에 기본으로 적혀 있는 `3fa85f64-...`는 단순 예시입니다. 이 값을 그대로 쓰면 에러가 나니, 반드시 내가 생성한 진짜 ID를 복사해서 사용해야 합니다.
2.  **지사 등록** (`POST /api/v1/branches/`): 
    - 지사 정보를 입력하고 실행합니다. 결과에서 나온 **`id`**를 복사하세요.
3.  **A/S 접수** (`POST /api/v1/requests/`): 
    - 위에서 복사한 **식당 ID**를 `restaurant_id` 칸에 넣고 접수합니다.
4.  **지사 할당** (`PATCH /api/v1/requests/{request_id}`):
    - 접수된 건의 ID와 복사한 **지사 ID**(`assigned_branch_id`)를 넣어 담당 지사를 지정해 봅니다.

### 3.3 팁: ID는 어디서 찾나요?
- 내가 만든 데이터의 ID를 잊어버렸다면, 언제든 **GET (조회)** API를 실행해 보세요. 목록 안에 모든 ID가 들어있습니다.

## ⚠️ 시행착오 및 트러블슈팅 가이드 (Lessons Learned)

개발 과정에서 실제로 겪었던 문제들과 그 해결책입니다. **비슷한 문제가 발생하면 이 섹션을 가장 먼저 확인하세요.**

### 1. 포트(Port) 접속 문제
- **현상**: 서버를 켰는데 `http://127.0.0.1:8000` 접속이 안 됨.
- **원인**: 윈도우 시스템에서 8000번 포트를 다른 용도로 선점하고 있는 경우가 많음.
- **해결**: 항상 **8888번** 포트를 사용합니다. (`--port 8888`)

### 2. ID(UUID) 혼동 주의
- **현상**: PATCH나 DELETE를 할 때 `404 Not Found`가 발생함.
- **원인**: **식당 ID**와 **접수(Request) ID**를 혼동하여 사용한 경우.
- **해결**: 
    - 식당 정보 수정 -> 식당 ID 사용
    - 수리 상태 변경 -> 접수(Request) ID 사용
    - *팁: Swagger의 GET 결과를 보고 내가 어떤 ID를 복사했는지 다시 확인하세요.*

### 3. Swagger 예시 데이터(3fa8...)의 함정
- **현상**: 데이터를 넣었는데 계속 에러가 남.
- **원인**: Swagger 입력창에 기본으로 적혀 있는 `3fa85f64-5717-4562-b3fc-2c963f66afa6`는 실제 데이터가 아닌 **모양만 보여주는 예시**입니다.
- **해결**: 반드시 `POST`나 `GET`을 통해 얻은 **진짜 데이터의 ID**를 넣어야 합니다.

### 4. JSON 문법 오류 (422 Error)
- **현상**: 실행 버튼을 눌렀는데 `422 Unprocessable Entity`가 뜸.
- **원인**: JSON 데이터 형식이 깨짐. (예: `{"data": {test}}` 처럼 따옴표가 없는 경우)
- **해결**: JSON 값에는 반드시 **큰따옴표(`" "`)**를 사용하고, 키와 값의 쌍을 지켜야 합니다. (예: `{"data": {"key": "test"}}`)

### 5. 서버 내부 에러 (500 Error / Unexpected token 'I'...)
- **현상**: 브라우저나 Admin UI에서 "Unexpected token 'I'..." 혹은 "Internal Server Error"가 뜸.
- **원인**: 서버 코드 실행 중 에러가 발생하여 서버가 다운되거나 멈춤.
- **해결**: **서버가 실행 중인 검은색 터미널 창**을 보세요. 빨간색으로 에러 원인(예: `NameError`, `ImportError`)이 적혀 있습니다. 그 메시지를 개발자에게 알려주세요.

### 6. 안전 삭제 코드 (Safety Code)
- **현상**: 지사 삭제(`DELETE`)가 안 됨.
- **원인**: `confirmation_code`가 틀렸거나 연관된 데이터가 있음.
- **해결**: 
    - 코드는 반드시 **`DELETE_CONFIRM`** (대문자)이어야 합니다.
    - 해당 지사에 배정된 수리 요청이 하나라도 있다면 지울 수 없습니다. (먼저 요청을 정리해야 함)

## 6. 개발 도구 및 환경 (Development Tools)

- **Python Launcher**: `python` 대신 `py` 명령어를 사용하면 윈도우 환경에서 파이썬 버전 충돌 없이 더 안정적으로 실행됩니다.

## 7. 외부 접속 및 공유 (External Access)

아버님이나 지인에게 결과물을 보여주어야 할 때는 아래 순서대로 진행하세요.

### Step 1: 서버를 0.0.0.0으로 실행
터미널에서 서버를 실행할 때 `--host 0.0.0.0` 옵션을 반드시 붙여야 합니다.
```powershell
py -m uvicorn app.main:app --reload --port 8888 --host 0.0.0.0
```

### Step 2: 인터넷 주소 만들기 (Localtunnel)
서버는 켜둔 채로, **새 터미널**을 하나 더 열어서 아래 명령어를 입력하세요.
```cmd
npx localtunnel --port 8888
```
1.  명령어를 입력하면 `your url is: https://xxxx-xxxx.loca.lt` 주소가 생성됩니다.
2.  해당 주소를 복사하여 공유하세요.
3.  접속 시 "Friendly Reminder" 화면이 나오면 **[Click to Continue]** 버튼을 누르면 됩니다.

> [!TIP]
> `npx` 명령어가 처음이라면 설치 여부를 물어볼 수 있습니다. `y`를 눌러 진행하시면 됩니다. (별도의 계정 가입이 필요 없어 가장 간편합니다.)
